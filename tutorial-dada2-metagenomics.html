Metagenomic Taxonomic Profiling With DADA2

A walkthrough of the downstream processing steps for amplicon sequencing data in R — from filtered FASTQ reads to a taxonomy table identifying bacterial and fungal community composition.

STEP 1

Filter & Trim Reads

Before any biological signal can be extracted, raw paired-end reads need quality filtering: trimming low-quality bases, removing reads with ambiguous base calls (N), and truncating to a consistent length. This step is critical for DADA2's downstream error model — inconsistent read quality or length propagates directly into inaccurate sequence variant calls. 

library(dada2)

path <- "path/to/fastq_files"
fnFs <- sort(list.files(path, pattern = "_R1.fastq", full.names = TRUE))
fnRs <- sort(list.files(path, pattern = "_R2.fastq", full.names = TRUE))
sample.names <- sapply(strsplit(basename(fnFs), "_R1"), `[`, 1)

filtFs <- file.path(path, "filtered", paste0(sample.names, "_F_filt.fastq.gz"))
filtRs <- file.path(path, "filtered", paste0(sample.names, "_R_filt.fastq.gz"))

out <- filterAndTrim(fnFs, filtFs, fnRs, filtRs,
                      truncLen = c(240, 160),
                      maxN = 0, maxEE = c(2, 2), truncQ = 2,
                      rm.phix = TRUE, compress = TRUE, multithread = TRUE)

STEP 2

Learn Error Rates & Denoise

DADA2's core strength is distinguishing true biological sequence variants from sequencing errors. It does this by first learning a run-specific error model from the data itself — how likely each type of base substitution is, as a function of quality score — then using that model to correct reads down to their most likely true sequence. This produces amplicon sequence variants (ASVs), a higher-resolution alternative to the older OTU-clustering approach, since ASVs resolve differences down to a single nucleotide rather than grouping reads by a similarity threshold. 

errF <- learnErrors(filtFs, multithread = TRUE)
errR <- learnErrors(filtRs, multithread = TRUE)

dadaFs <- dada(filtFs, err = errF, multithread = TRUE)
dadaRs <- dada(filtRs, err = errR, multithread = TRUE)

STEP 3

Merge Pairs & Build the Sequence Table

Forward and reverse reads are merged where their overlapping region matches, reconstructing the full amplicon sequence. The result is assembled into a sequence table — samples as rows, unique ASVs as columns, with read counts as values. Chimeras (artificial sequences formed when two different DNA templates get spliced together during PCR) are then identified and removed, since they don't represent real organisms and would otherwise inflate apparent diversity. 

mergers <- mergePairs(dadaFs, filtFs, dadaRs, filtRs, verbose = TRUE)

seqtab <- makeSequenceTable(mergers)

seqtab.nochim <- removeBimeraDenovo(seqtab, method = "consensus",
                                     multithread = TRUE, verbose = TRUE)

STEP 4

Assign Taxonomy

Each ASV is then classified against a reference database using a naive Bayesian classifier — SILVA for bacterial/archaeal 16S rRNA sequences, or UNITE for fungal ITS sequences, depending on which marker gene was sequenced. The output assigns each ASV a taxonomic label from kingdom down to genus (and species, where reference resolution allows) — turning a table of anonymous DNA sequences into an interpretable profile of which bacterial or fungal taxa are present in each sample, and at what relative abundance. 

taxa <- assignTaxonomy(seqtab.nochim,
                        "silva_nr99_v138_train_set.fa.gz",
                        multithread = TRUE)

# For fungal ITS data, the UNITE reference set is used instead:
# taxa <- assignTaxonomy(seqtab.nochim, "unite_reference_set.fa.gz", multithread = TRUE)

From here, the ASV table and taxonomy assignments can be carried into diversity analysis (alpha/beta diversity), differential abundance testing between conditions, or visualization of community composition across samples.
