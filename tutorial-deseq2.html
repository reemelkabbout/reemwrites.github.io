Differential Expression Analysis With DESeq2

A walkthrough of RNA-seq differential expression analysis in R — from a raw count matrix to a filtered, annotated list of genes that differ significantly between two conditions.

STEP 1

Load Counts & Define the Experimental Design

DESeq2 needs two inputs: a count matrix (raw read counts, genes as rows and samples as columns) and a sample metadata table describing which group each sample belongs to. It's important that the count matrix contains raw, un-normalized counts — DESeq2 performs its own normalization internally, so pre-normalized data (like TPM or FPKM) should not be used here. 

library(tidyverse)
library(biomaRt)
library(DESeq2)
library(dplyr)

# Load sample metadata (sample IDs + group assignment)
col <- read.csv("sample_metadata.csv", check.names = FALSE)

# Load raw count matrix (genes x samples)
counts <- read.table("raw_counts.txt")

# Select only the samples included in this comparison
counts <- dplyr::select(counts, all_of(col$Sample))

# Define the grouping variable — the reference level (e.g. "Control")
# should be listed first
col$Group <- factor(col$Group, levels = c("Control", "Disease"))

# Sanity check: sample order must match between counts and metadata
all(colnames(counts) == col$Sample)

STEP 2

Run DESeq2

DESeqDataSetFromMatrix() combines the counts and metadata into a single object, with the design formula specifying which variable to test — here, ~Group means we're comparing gene expression between the Control and Disease groups. The DESeq() function then runs the full pipeline in one step: it estimates size factors to normalize for sequencing depth, estimates gene-wise dispersion (variability), and fits a model to test each gene for a significant difference in expression between groups. 

dds <- DESeqDataSetFromMatrix(countData = round(counts),
                               colData = col,
                               design = ~Group)

dds <- DESeq(dds)

result <- results(dds)
result <- as.data.frame(result)

STEP 3

Filter for Significant Genes

The results table includes a p-value and an adjusted p-value (padj) for every gene. Because thousands of genes are tested simultaneously, raw p-values need correction for multiple testing — DESeq2 uses the Benjamini-Hochberg method by default, controlling the false discovery rate. Filtering on padj < 0.05 keeps only genes with a statistically significant difference in expression between groups, after that correction. 

# Remove genes with missing values (e.g. due to low counts)
filter1 <- result[complete.cases(result), ]

# Keep only statistically significant genes
filter2 <- filter1[filter1$padj < 0.05, ]

STEP 4

Annotate Gene IDs

DESeq2 output is indexed by Ensembl gene ID, which isn't very readable on its own. biomaRt queries the Ensembl database directly to map each ID to its gene symbol and description, making the results interpretable. Pinning a specific Ensembl release (rather than always using the latest) keeps annotations consistent and reproducible across analyses. 

# Connect to a specific, versioned Ensembl release for reproducibility
ensembl <- useEnsembl(biomart = "ensembl", version = 104)
ensembl <- useDataset("hsapiens_gene_ensembl", mart = ensembl)

filter2 <- rownames_to_column(filter2, var = "ensgene")

annotation <- getBM(attributes = c("ensembl_gene_id",
                                    "external_gene_name",
                                    "description"),
                     filters = "ensembl_gene_id",
                     values = filter2$ensgene,
                     mart = ensembl)

annotated_results <- left_join(filter2, annotation,
                                by = c("ensgene" = "ensembl_gene_id"))

From here, the annotated table can be further filtered — for example by effect size (log2FoldChange) — or carried forward into pathway enrichment or connectivity-mapping analyses.
